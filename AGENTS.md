# 작업 가이드라인

이 저장소를 수정하는 모든 봇/모델은 아래 규칙을 따른다.

## 기본 원칙

- 작업 전에 `git status`와 관련 README를 확인하고, 현재 변경사항을 보존한다.
- 요청 범위를 벗어난 리팩터링, 파일 이동, 포맷 일괄 변경을 하지 않는다.
- 비밀값(텔레그램 토큰, chat ID, API 키, 계좌정보)을 코드·설정·로그·커밋에 넣지 않는다.
- 운영 데이터와 개인 설정은 저장소에 추가하지 않는다. `.gitignore`를 수정할 때도 이 원칙을 유지한다.
- 변경 이유와 영향 범위를 커밋 메시지에 명확히 적는다.
- 모든 업데이트마다 `CHANGELOG.md`에 날짜, 변경 파일, 구체적인 수정 내용, 동작/위험 영향, 검증 결과를 기록한다.
- 단순한 "버그 수정" 같은 표현만 쓰지 말고, 무엇이 어떻게 바뀌었는지 재현 가능한 수준으로 작성한다.
- 코드 변경과 해당 `CHANGELOG.md` 변경은 반드시 같은 커밋에 포함한다.

## 검증

- Python 변경 후 `python -m py_compile <변경 파일>`을 실행한다.
- 네트워크/API 호출은 실제 주문·송금으로 이어지지 않는지 확인한다.
- 기본 동작(`--once`)을 우선 검증하고, 데몬/운영 프로세스는 임의로 시작·중지하지 않는다.
- 실패한 검증과 검증하지 못한 부분은 작업 결과에 기록한다.

## 이 봇의 안전 경계

- `upbit_observer.py`는 Upbit 공개 시세 조회와 알림만 담당한다.
- 주문, 매도, 출금, 개인키 인증 기능을 추가하지 않는다.
- 주식 봇의 자격증명·설정·상태 파일과 이 봇의 자격증명·설정·상태를 섞지 않는다.
- 기능 변경 시 README와 실행 예시를 함께 갱신한다.
